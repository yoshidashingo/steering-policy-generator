# 出力構造パターン (Output Structure Patterns)

## 目的

このファイルは、生成されるすべての提案書成果物が従うべき標準化された構造パターンを定義します。これらのパターンにより、提案書ドキュメント間の一貫性が確保され、ワークフローおよび人間のレビュー担当者の双方にとって出力が予測可能になります。

---

## 提案書成果物パターン

### RFP 概要パターン

```markdown
# RFP 概要 — [RFP タイトル]

## RFP 全体像
- **発行機関**: [名称]
- **RFP 番号**: [番号/ID]
- **発行日**: [日付]
- **提出期限**: [日時]
- **質問受付期限**: [日付（該当する場合）]
- **契約タイプ**: [FFP / T&M / CPFF / その他]
- **推定金額**: [範囲または金額（開示されている場合）]

## 業務範囲
[調達対象の簡潔な説明]

## 文書構成
| セクション | タイトル | ページ数 | 主な内容 |
|-----------|---------|---------|---------|
| [番号] | [タイトル] | [ページ数] | [概要] |

## 主要日程
| イベント | 日付 |
|---------|------|
| [イベント] | [日付] |

## 提出要件
- **形式**: [電子 / 紙媒体 / 両方]
- **ページ制限**: [ページ数（該当する場合）]
- **部数**: [必要部数]
- **提出方法**: [メール / ポータル / 郵送]

## 評価方法
[提案書の評価方法の概要]
```

### 提出チェックリストパターン

```markdown
# 提出チェックリスト — [RFP タイトル]

## 提出前確認

### 文書の完全性
- [ ] エグゼクティブサマリーが含まれている
- [ ] 技術提案が完成している
- [ ] マネジメント提案が含まれている（必要な場合）
- [ ] 価格フレームワークが含まれている（必要な場合）
- [ ] すべての付録が添付されている
- [ ] コンプライアンスマトリクスが含まれている

### 形式の準拠
- [ ] ページ制限を確認: [X] ページ（上限: [Y]）
- [ ] フォントおよびフォーマットが RFP 要件に準拠している
- [ ] ヘッダー/フッターが指定通りである
- [ ] ページ番号が正しい
- [ ] 目次が正確である

### 内容の確認
- [ ] すべての必須要件に対応している
- [ ] コンプライアンスマトリクスが完成している
- [ ] 相互参照が確認済みである
- [ ] プレースホルダーテキストが残っていない
- [ ] 会社名およびブランディングが一貫している

### 提出ロジスティクス
- [ ] 提出形式を確認: [電子/紙媒体]
- [ ] 提出方法: [方法]
- [ ] 期限: [日時]
- [ ] 送付先: [名前/住所]
- [ ] 部数: [部数]

## 最終承認
- [ ] 承認権限者がレビュー済みである
- [ ] 法務レビューが完了している（必要な場合）
- [ ] 価格が承認権限者により承認されている
```

## 完了メッセージパターン

### ステージ完了（標準 2 選択肢）

```markdown
---

### レビューが必要です

**[ステージ名] が完了しました。** 生成された内容の概要は以下の通りです：

- [出力 1 の説明]
- [出力 2 の説明]

### 次のステップ

以下のいずれかを選択してください：

**A) 変更を依頼する** — フィードバックに基づいて修正します
**B) [次のステージ名] に進む** — 次のステージへ進みます

---
```

### フェーズ完了

```markdown
---

### フェーズ完了: [フェーズ名]

**[フェーズ名] フェーズのすべてのステージが完了しました。**

**出力の概要:**
- [主要な出力 1]
- [主要な出力 2]

### 次のステップ

次のフェーズは **[次のフェーズ名]** です: [簡潔な説明]。

**A) [次のフェーズ名] に進む** — ワークフローを続行します
**B) [現在のフェーズ名] の出力を確認する** — 先に進む前に確認します

---
```

### ワークフロー完了

```markdown
---

### 提案書完成

**[RFP タイトル] の提案書が生成され、レビューが完了しました。**

**最終成果物:**
- ディレクトリ: `proposal-docs/`
- コンプライアンススコア: [X]% の要件に対応
- 品質スコア: [X]/22 の品質次元

**内容:**
- エグゼクティブサマリー
- 技術提案
- [該当する追加セクション]
- コンプライアンスマトリクス
- 提出チェックリスト

### 次のステップ

1. `proposal-docs/` 内の提案書全体を確認する
2. 主題専門家 (SME) による内部レビューを実施する
3. RFP の指示に従い最終提出パッケージを準備する
4. 期限前に提出する

---
```

## 実装ガイドライン

1. **パターンに正確に従うこと**: 正当な理由なく確立されたパターンから逸脱しないこと
2. **すべてのフィールドを含めること**: オプションのフィールドであっても存在させること（該当しない場合は N/A と記載）
3. **一貫性を維持すること**: すべてのインスタンスで同じパターン形式を使用すること
4. **正確に相互参照すること**: すべてのセクション参照は実際の内容に解決されなければならない
