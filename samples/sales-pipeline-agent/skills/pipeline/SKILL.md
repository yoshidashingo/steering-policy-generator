# Pipeline Management Skill — コアルール & ワークフローサマリー

## スキル概要

営業パイプライン管理に特化したコアルール集。4ソース統合巡回型エージェントの行動原則、判定ルール、品質基準を定義する。

---

## Core Rules（5つ）

### Rule 1: 多層判定の必須適用（Pitfall 1対策）

商談の停滞・異常を判定する際は、Salesforceの単一ソースに依存せず、Gmail・Slack・Calendarのアクティビティを必ず加味する。

**適用条件**: 停滞商談検出（S4）、ヘルススコア算出（A1）
**判定式**:
```
停滞 = Salesforce滞留日数 >= 閾値
       AND Gmail/Slack/Calendar直近アクティビティなし
```

**違反例**: Salesforceのステージ滞留日数のみで停滞判定し、Slackで活発に交渉中の商談にアラートを送信する。

---

### Rule 2: 権限境界の厳守（Pitfall 2対策）

エージェントは低リスクアクション（内部記録・通知・下書き生成）のみ自動実行する。顧客向け送信・ステージ更新・金額変更は必ず担当者承認を得てから実行する。

**自動実行可能**:
- Salesforce Activityへのログ記録
- Slack/メール下書き生成
- 内部Slack通知送信

**承認制必須**:
- 顧客向けメール送信
- Salesforceステージ更新
- 金額・クロージング日の変更

---

### Rule 3: 1商談1アクション原則（Pitfall 3対策）

同一商談に複数の検出結果がある場合、最高優先度の1件のみを主アクションとし、他をコンテキストに付加する。承認リクエストは同一担当者あて最大3件をバッチ化する。

**重複通知抑制**:
- 未返信リマインダー: 前回送信から4時間以内は再送しない
- 期限接近通知（P2以下）: 前回送信から48時間以内は再送しない
- 期限接近（P1: 1日以内）: 重複抑制適用しない（毎サイクル通知）

---

### Rule 4: APIガバナーリミット保護（Pitfall 4対策）

Salesforce APIの日次使用率を毎サイクル開始時に確認し、使用率に応じてモードを切り替える。差分取得（全量取得禁止）とBulk API 2.0優先使用でAPI消費を最小化する。

**モード切り替え閾値**:
- < 80%: 通常モード（1時間サイクル）
- 80%-95%: セーフモード（2時間サイクル、管理者通知）
- >= 95%: 読み取り専用モード（DML操作停止）

---

### Rule 5: ソース間矛盾の透明な処理（Pitfall 5対策）

複数ソース間でデータが矛盾する場合（例: Salesforce停滞 + Slack活発）、矛盾フラグを付与して自動処理から除外し、人間確認依頼アクションとして登録する。矛盾を隠蔽・無視することは禁止。

**矛盾パターン**:
- `conflict_stale_but_active`: 停滞判定だが他ソースでアクティブ
- `conflict_post_close_activity`: クローズ後に活動検出
- `conflict_activity_not_recorded`: Gmail/Calendar活動がSalesforceに未記録
- `conflict_amount_discrepancy`: 金額変更言及がSalesforceに未反映

---

## Workflow Summary

```
【日中サイクル: 毎時実行】
S1 スケジュール確認
  └─ 日中モード
       S2 ソース接続確認（4ソース並行）
       S3 差分データ収集 + 統合ビュー構築
       S4 変更・異常検出（7種のルール適用）
       A1 ヘルススコア算出
       A2 アクション生成
       A3 優先度付け
       A4 権限分類（自動実行 / 承認制）
       D1 自動実行（低リスクアクション）
       D2 承認リクエスト送信（高リスクアクション）
       D3 承認処理・実行
       D4 ループ制御（残件チェック、最大5ループ）
       D5 ロールバック検証
       R1 日次サマリー生成・配信
       R3 監査ログ確定

【夜間サイクル: 20:00以降】
S1 スケジュール確認
  └─ 夜間モード
       R2 翌日準備（会議サマリー・翌朝アラート予約）
       R3 監査ログ確定
```

## 検出種別と優先度

| 優先度 | 検出種別 | 閾値 |
|--------|---------|------|
| P1 Critical | 期限接近（1日以内） | CloseDate <= 1日 |
| P1 Critical | 矛盾フラグ（人間確認） | conflict_flags != [] |
| P2 High | 停滞商談（ヘルスRed） | ステージ別閾値超 |
| P2 High | 期限接近（2-7日） | CloseDate <= 7日 |
| P2 High | 金額変動 | 変動率 >= 10% |
| P3 Medium | 未返信メール | 3営業日以上未返信 |
| P3 Medium | 会議準備 | 24時間以内の予定 |
| P3 Medium | 未追跡リード | Salesforce未登録ドメイン |
| P4 Low | ステータス未反映 | 24時間以上未記録 |
