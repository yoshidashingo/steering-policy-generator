# セッション継続性

## 目的

このファイルは、提案書作成ワークフローがセッション間で状態を維持し、中断後にシームレスに再開できるようにする方法を定義します。セッション継続性により、作業が失われず、中断箇所から正確にワークフローを再開できます。

---

## 状態追跡

### 提案書状態ファイル（`proposal-state.md`）

状態ファイルはワークフロー進捗の唯一の信頼できる情報源です。ワークフロー開始時に作成され、各ステージ完了後に更新されます。

**状態ファイルの形式**：

```markdown

- **RFPタイトル**: [RFP文書からのタイトル]
- **発注組織**: [組織名]
- **提出期限**: [日付]
- **RFP種類**: [ITサービス / コンサルティング / 建設 / 等]
- **競争コンテキスト**: [随意契約 / 競争入札]
- **作成日時**: [ISOタイムスタンプ]
- **最終更新日時**: [ISOタイムスタンプ]
```

## セッション再開手順

### ステップ1: 前回セッションの検出
1. 既存の `proposal-docs/proposal-state.md` を確認
2. 見つかった場合: 再開フローを開始
3. 見つからない場合: 新しいワークフローを開始（ウェルカムメッセージを表示）

### ステップ2: 状態の読み込み
1. `proposal-state.md` を完全に読み込む
2. 現在のフェーズ、ステージ、ステップを特定
3. 状態ファイルの整合性を確認（欠損フィールドなし、有効なタイムスタンプ）

### ステップ3: 成果物の検証
1. 完了済みステージの成果物がすべて実際に存在することを確認
2. 各完了済みステージについて、出力ファイルが存在し空でないことを確認
3. 成果物がないのにステージが完了とマークされている場合: 不整合をフラグ付け

### ステップ4: コンテキストの読み込み
1. 現在のステージに関連する完了済みステージの成果物を読み込む
2. コンテキストのためにaudit.mdの最新エントリを読み込む
3. 回答待ちの質問ファイルを読み込む

### ステップ5: 再開サマリーの提示

```markdown
# セッション再開

**RFP**: [RFPタイトル]
**最終アクティブ**: [最終更新タイムスタンプ]

## 進捗サマリー
- 分析: [完了 / 進行中 / 未着手]
- 計画: [完了 / 進行中 / 未着手]
- 執筆: [完了 / 進行中 / 未着手]
- レビュー: [完了 / 進行中 / 未着手]

## 現在位置
- **フェーズ**: [フェーズ名]
- **ステージ**: [ステージ名]
- **状態**: [セッション終了時の進行内容]

## 続行の準備ができています

**A) [現在位置]から再開する** — 中断箇所から続行
**B) 前回の作業を確認する** — 続行前に完了済みステージをレビュー
```

## 状態更新ルール

### 更新するタイミング
1. **ステージ開始時**: ステージを「進行中」にマーク
2. **ステージ完了時**: ステージを「完了」にマーク（タイムスタンプ付き）
3. **ステージスキップ時**: ステージを「スキップ」にマーク（理由付き）
4. **主要な決定時**: 主要な決定事項セクションに追加
5. **条件付き評価時**: 実行/スキップの決定を理由付きで記録

### 更新方法
1. 現在の `proposal-state.md` を読み込む
2. 関連するフィールドを更新する
3. 「最終更新日時」タイムスタンプを更新する
4. ファイルに追記する（完全に上書きしない）

## 再開時のエラー回復

### 状態ファイルが見つからない
- `proposal-docs/` ディレクトリが存在するか確認
- 成果物は存在するが状態ファイルがない場合: 成果物から状態を再構築
- 成果物もない場合: 新しいワークフローを開始

### 状態ファイルが破損している
- バックアップを作成: `proposal-state.md.backup`
- ユーザーにどのフェーズにいるか尋ねる
- 既存の成果物から状態を再構築
- 検証された位置から再開

### 完了済みステージの成果物が見つからない
- ステージを未完了としてマーク
- ステージを再実行するか、手動で情報を提供するかユーザーに尋ねる
- 回復アクションをaudit.mdに記録

包括的なエラーハンドリング手順は `common/error-handling.md` を参照。

---

## 実装ガイドライン

1. **状態を即座に更新する**: 状態の更新を後のインタラクションに延期しない
2. **再開前に検証する**: 続行前に常に成果物の整合性を確認する
3. **ユーザーコンテキストを保持する**: 意味のある再開のために状態に十分な詳細を含める
4. **すべての再開を記録する**: セッション再開をタイムスタンプ付きでaudit.mdに記録する
